# ANTI-SLOP — Non-fiction edition

Rules injected into every drafting prompt to prevent common AI writing pathologies.

---

## Prohibited words and phrases

Never use these:
- "delve into"
- "in today's rapidly evolving landscape"
- "it is important to note that"
- "furthermore" / "moreover" (use "also" or restructure)
- "in conclusion" (the reader knows it's the conclusion)
- "as we have seen"
- "it is worth noting that"
- "on the other hand" (lazy — show the contrast)
- "in order to" (use "to")
- "due to the fact that" (use "because")

## Structure rules

- Opening sentences: no "This essay will..." / "In this paper, I will..."
  Start with a claim, a question, or a scene — per the style profile.

- Paragraphs: minimum 2 sentences, maximum 6. Vary rhythm.

- Transitions: earn them. Don't use "Firstly..." / "Secondly..." unless the style profile explicitly calls for enumeration (e.g., Policy Brief).

- Conclusions: don't summarize. Advance the argument or leave the reader with a new thought.

## Voice rules

- Default to active voice. Passive only when the style profile explicitly allows it (Academic).

- Cut hedge words: "arguably," "somewhat," "relatively," "generally," "may be." If you're not sure, say so directly or qualify with evidence.

- Numbers under 10: spell out. Numbers 10+: digits. Exceptions: percentages always digits (8%).

- No rhetorical questions unless the style profile explicitly permits them (Personal Essay).

## Evidence rules

- Every factual claim needs a source citation: [src_01]
- Don't cite a source you haven't been given
- If a source is weak, say so ("This is based on limited evidence from [src_01]...")
- Don't over-cite — one source per claim, not three sources in one sentence
